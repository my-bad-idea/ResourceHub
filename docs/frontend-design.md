## 前台设计文档

### 1. 文档目的与适用范围

本文件面向前端开发、设计和产品同学，描述 ResourceHub 的页面结构、路由、布局与关键交互逻辑。重点是“页面长什么样、怎么交互”，不重复实现细节与 API 细节。

### 2. 页面与路由概览

前端使用 Hash 路由，所有页面均挂载在同一 SPA 中，通过 `window.location.hash` 和 `hashchange` 事件切换：

| Hash 路径 | 页面组件 | 说明 |
|-----------|----------|------|
| `#/setup` | `SetupPage` | 系统首次初始化向导 |
| `#/login` | `LoginPage` | 登录页 |
| `#/register` | `RegisterPage` | 注册页（是否开放由配置控制） |
| `#/forgot-password` | `ForgotPasswordPage` | 忘记密码页 |
| `#/reset-password?token=xxx` | `ResetPasswordPage` | 重置密码页 |
| `#/` | `HomePage` | 首页 / 资源导航 |
| `#/admin` 及子路径 | `AdminPage` | 后台管理（仅管理员） |

路由守卫规则（高层）：

- 若 `GET /api/auth/init-status` 返回 `initialized: false`，无论当前 Hash 为何，前端应导航到 `#/setup`。
- 访问 `#/admin` 以及其子视图时，需要当前用户为管理员，否则重定向回 `#/`。

### 3. 布局与全局导航

#### 3.1 顶部 Header

所有业务页（包括首页与后台）均使用统一的 `Header` 组件，主要区域包括：

- **左侧**：
  - 系统 Logo：点击返回首页 `#/`。
  - 系统名称：读取自公开配置 `siteTitle`。
- **中间**：
  - 全局搜索框：占据中间主要宽度，支持快捷键聚焦（如 `Ctrl+K` / `⌘K`）。
  - 搜索范围：资源名称、描述、URL 和标签。
  - 在后台视图中也可以复用搜索框（是否启用由 `showSearch` 控制）。
- **右侧**：
  - 主题切换入口：弹出菜单选择浅色/深色/跟随系统。
  - 用户区域：
    - 未登录：显示“登录”按钮，点击跳转 `#/login`。
    - 已登录：显示头像（首字母）+ 用户名，点击展开下拉菜单：
      - 个人信息（打开 `ProfileModal`）。
      - 修改密码（打开 `ChangePasswordModal`）。
      - 后台管理入口（仅管理员时显示，跳转 `#/admin`）。
      - 退出登录。

Header 使用 `position: fixed` 吸附在页面顶部，配合 `AppLayout` 通过 CSS 变量 `--app-header-height` 预留内容区顶部空间。

#### 3.2 整体布局（AppLayout）

大部分业务页面通过 `AppLayout` 包裹，提供：

- 顶部固定 Header（见上）。
- 左侧 Sidebar（在桌面端展示，在窄屏时堆叠到内容上方或折叠为 Drawer）。
- 右侧主内容区域：
  - 支持按照视口宽度调整内边距。
  - 支持不同页面配置 `contentPaddingTop` 与 `headerVariant`，实现首页顶部渐变背景等视觉效果。
- 全局浮层与 Modal：
  - Toast 通知容器。
  - Email 预览弹窗。
  - 个人信息弹窗。
  - 修改密码弹窗。

Sidebar 在宽屏下固定宽度，在窄屏下与内容区垂直排列或被折叠，通过内部状态控制展开/收起。

#### 3.3 首页布局（HomePage）

首页主要区域由 `AppLayout` + `Sidebar` + 主内容区组成：

- **Sidebar**：
  - 类别导航区：
    - 顶部展示“全部”以及各个类别。
    - 每个类别项显示名称和资源数量角标（数量根据当前用户可见范围统计）。
    - 当前选中类别高亮显示。
  - 标签云区：
    - 展示所有标签，多选，选中标签高亮。
    - 若标签较多，提供“展开更多/收起”逻辑，保证 Sidebar 不会过高。
  - 快速访问区（仅登录用户）：
    - “我的收藏”、“最近访问”、“我创建的”三项。
    - 点击后主内容区切换数据来源，与类别/标签筛选互斥。
- **主内容区**：
  - 顶部统计 & 概览：
    - 显示资源数量、类别数量以及“我的资源”等指标卡片。
    - 显示访问统计卡片，总访问量、近 30 日访问量、近 24 小时访问量。
  - 工具栏：
    - 视图切换按钮：卡片、列表、时间轴三种。
    - 排序方式选择：按热度 / 按创建时间 / 按更新时间。
    - 右侧“新增资源”按钮（仅登录用户可见），点击打开 `ResourceFormModal`。
  - 资源列表区域：
    - 根据视图模式不同渲染：
      - 卡片视图：多列卡片网格。
      - 列表视图：表格样式行列表。
      - 时间轴视图：按月份分组的时间轴布局。
    - 支持空状态：
      - 无数据时显示图标 + 文案 + 引导操作按钮（如“新增资源”或“清空筛选”）。

### 4. 关键交互行为

#### 4.1 搜索与筛选联动

首页的搜索、类别、标签与快速访问之间有以下联动规则：

- 点击类别：
  - 设置当前选中类别。
  - 清空已选标签。
  - 取消快速访问筛选。
- 点击标签（支持多选）：
  - 在当前类别/搜索条件下进一步 AND 过滤。
  - 取消快速访问筛选。
- 点击快速访问项（收藏/历史/我创建的）：
  - 切换资源数据来源为对应预加载的数据集。
  - 清空类别和标签筛选。
  - 搜索词保留，在当前快速访问结果内二次过滤。
- 搜索框输入：
  - 在当前筛选范围内（类别/标签/快速访问）进行搜索。
- “清空筛选”按钮：
  - 一键清空搜索词、类别、标签和快速访问筛选。

#### 4.2 收藏、访问与私有资源展示

- 收藏：
  - 资源卡片或列表项右上角显示心形按钮。
  - 已收藏时心形实心高亮，点击再次取消收藏。
  - 未登录用户点击收藏时，弹出 Toast 提示“请登录后操作”，不进行后端调用。
- 访问记录：
  - 点击卡片或列表行（非收藏按钮区域）在新标签页打开资源 URL。
  - 登录用户点击后会记录访问历史，用于“最近访问”视图和访问统计。
- 私有资源标识：
  - 仅资源 owner 和管理员可见。
  - 在卡片或列表中通过“锁”图标或“私有”角标进行标识。

#### 4.3 弹窗与表单交互

常见弹窗包括：

- `ResourceFormModal`（新增/编辑资源）：
  - 字段：名称、URL、类别（含新增类别）、可见性、Logo URL、描述、标签、启用状态。
  - 校验：必填字段规则、URL 格式、描述与标签长度等。
  - 提交中按钮显示 loading，防止重复提交。
- `ProfileModal`（个人信息）：
  - 显示名称和邮箱可编辑，用户名与角色只读。
  - 保存成功后刷新本地用户信息。
- `ChangePasswordModal`（修改密码）：
  - 字段：当前密码、新密码、确认新密码。
  - 提供密码强度指示条。
- `EmailPreviewModal`（邮件预览）：
  - 当任意接口响应携带 `emailPreview` 字段时自动弹出。
  - 展示收件人、主题、正文内容，方便在 Mock 模式下查看邮件内容。

所有弹窗都应支持点击遮罩或关闭按钮关闭，并在关闭时清理内部表单状态。

### 5. 前端状态与主题处理（高层）

#### 5.1 全局状态（AppContext）

`AppContext` 使用 `useReducer` 管理以下状态（高层视角）：

- 认证信息：当前用户、token。
- 数据缓存：资源、类别、标签、公开配置、快速访问数据（收藏、历史、我创建的）。
- UI 状态：主题（`light` / `dark` / `system`）、Toast 队列、当前激活 Modal、邮件预览内容。
- 筛选状态：搜索词、选中类别、选中标签、快速访问筛选、视图模式、排序方式。

初始加载流程：

1. 请求 `/api/auth/init-status`，若未初始化则导航到 `#/setup`。
2. 若已初始化：
   - 请求 `/api/config/system` 获取站点配置。
   - 从 `sessionStorage` 读取 token，若存在则请求 `/api/auth/me` 获取当前用户。
   - 并行加载类别、标签以及快速访问数据（收藏、历史、我创建的）。
   - 解析当前 Hash 决定初始页面。

#### 5.2 主题与本地存储

- 主题值存储在 `localStorage` 的 `rh_theme` 键下。
- 取值范围为：`'light'`、`'dark'`、`'system'`。
- 应用时通过 `document.documentElement.classList.toggle('dark', ...)` 切换，配合 CSS 变量控制色板。
- JWT token 仅存储在 `sessionStorage` 中，关闭标签页即失效。

### 6. 后台管理界面设计

后台通过 `AdminPage` + `AdminLayout` 实现整体布局：

- 左侧为后台专属侧边栏，包含 5 个 Tab：
  - 类别管理（`categories`）。
  - 标签管理（`tags`）。
  - 用户管理（`users`）。
  - 系统配置（`config`）。
  - 邮件服务配置（`email`）。
- 右侧内容区域根据当前子路由渲染对应管理模块组件：
  - `AdminCategories`、`AdminTags`、`AdminUsers`、`AdminConfig`、`AdminEmail`。

各后台模块遵循以下共性设计：

- 列表视图采用表格样式，提供基础排序与分页（如每页 20 条）。
- 新增和编辑操作通过 Modal 完成，字段布局与前台表单风格一致。
- 删除等危险操作需弹出确认对话框。
- 若数据为空时，显示合理的空状态提示，包含“新增”入口。

### 7. 设计规范摘要（视觉层面）

仅保留前端实现时需要记住的少数关键点，其余细节可参考原始视觉规范：

- 颜色与主题：
  - 所有组件使用 CSS 变量（`--bg-*`、`--text-*`、`--brand`、`--border`、`--danger`、`--success` 等），不得直接写死色值。
  - 浅色模式与深色模式通过 `.dark` 类切换变量值。
- 布局：
  - Header 高度约 64–72 px，固定在顶部。
  - 桌面端 Sidebar 宽度约 220 px，使用 `sticky` 保持随滚动固定。
  - 主内容区内边距在中大屏保持 24 px 左右，小屏略收窄。
- 动效：
  - 交互动画大多在 150–300 ms 范围，统一使用平滑的 `cubic-bezier(0.4, 0, 0.2, 1)`。
  - 收藏按钮、Toast、Modal 出入场需有基础的缩放/位移动画，保证反馈明确但不过分夸张。
- 响应式：
  - ≥ 1280px：完整桌面布局，卡片 3–4 列，Sidebar 固定。
  - 768–1279px：Header 和工具栏允许换行，Sidebar 可改为堆叠。
  - < 768px：Sidebar 堆叠到顶部或折叠为 Drawer，资源列表自动换列或改为单列。

以上前台设计文档与现有实现保持对齐，后续如对交互或布局做重大改动，应同步更新本文件。
